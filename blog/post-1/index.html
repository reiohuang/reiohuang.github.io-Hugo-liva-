第 4 章 展示你的個人網站

單元1 - 發佈個人網站到 Github

使用 Hugo 產生個人網站的程式碼： 之前是輸入 hugo server 指令去運行 Hugo 在個人電腦上，如果要使用 Hugo 來產生個人網站的程式碼，只要輸入 hugo 即可。 ![[Pasted image 20220313234121.png]]

回到 Finder 上，會發現在 example Site 裡面多了一個 public 的資料夾，就是 Hugo 新產生出來的個人網站的程式碼。 ![[Pasted image 20220313234331.png]]

複製 public 裡面全部的檔案跟資料夾放到==本地端== reiohuang.github.io 資料夾的 repository 裡，最後再把它上傳到遠端 Github 的 reiohuang.github.io 資料夾上。 ![[Pasted image 20220313234601.png]]

git 初始化本地端 reiohuang.github.io 資料夾準備與遠端的 repository 綁定：

git init
git add .
git ci -m 'my first commit'

在 Github 中點擊 New repository，創建一個新的 repository。如果要使用 Github Pages 來架設個人網站，repository 的名字有命名規則，Github 帳號.github.io。 ![[Pasted image 20220313223715.png]]

本地端的資料夾與遠端儲存庫綁定，並 push 資料到 GitHub 遠端。 ![[Pasted image 20220314000215.png]]

成功上傳到 GitHub ![[Pasted image 20220314000653.png]]

開啟瀏覽器，網址列輸入 reiohuang.github.io，即可看到 Github Pages 已經成功地幫我們把 Hugo 的個人網站程式碼轉換成網站。

每次新增、修改文章時，記得最前面要貼上 Hugo 的文章設定，才能正常顯示。

單元2 - 添加評論功能

需要依靠其他的第三方網站才能達成，原理是因為 Github Pages 他是屬於靜態網頁，而評論功能是需要後端資料庫儲存，如果只使用 Github Pages 是沒有辦法加入評論功能。

進入 Hugo 首頁，Search the Docs 輸入 comments，點擊第一個連結，Hugo 預設支持的評論功能網站是 Disqus，所以要先進到 Disqus 這個網站申請一個帳號，在 Disqus 創建一個評論功能之後再回頭修改程式碼，把 Disqus 的帳號跟個人網站連結在一起。

注意：雖然 Hugo 預設支持的評論功能是 Disqus 網站，但是有的 Hugo 主題可能沒有開啟這個功能，所以要先檢查主題有沒有辦法加入評論功能。

在 config.toml 設定檔裡面有找到 disqusShortname 的設定的話，就表示你使用的 Hugo 主題是支持 Disqus 評論功能，把申請的 Disqus 評論功能帳號填在這裡即可。

![[Pasted image 20220314232444.png]]

如果設定檔裡面沒有出現 disqus Shortname，表示 Hugo 主題目前是沒有辦法開啟評論功能。如果主題沒有辦法開啟評論功能，可以去修改這個 Hugo 的主題，讓他去支持評論功能，不過需要有 html 的程式碼基礎才能改得了。

進入 Hugo 首頁，Search the Docs 輸入 disqus，點擊第一個連結，首先要先把 Hugo 內建的 Disqus html 的 template 放到文章的下方…未完待續。
